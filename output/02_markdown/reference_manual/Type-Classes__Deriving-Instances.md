#  10.4. Deriving Instances[🔗](find/?domain=Verso.Genre.Manual.section&name=deriving-instances "Permalink")
Lean can automatically generate instances for many classes, a process known as _deriving_ instances. Instance deriving can be invoked either when defining a type or as a stand-alone command.
syntaxInstance Deriving (Optional)
As part of a command that creates a new inductive type, a ``Lean.Parser.Command.declaration : command```deriving` clause specifies a comma-separated list of class names for which instances should be generated:

```
[optDeriving](Type-Classes/Deriving-Instances/#Lean___Parser___Command___optDeriving-next) ::=
    (deriving derivingClass,*)?
```

syntaxStand-Alone Deriving of Instances
The stand-alone ``Lean.Parser.Command.deriving : command``[`deriving`](Type-Classes/Deriving-Instances/#Lean___Parser___Command___deriving-next) command specifies a number of class names and subject names. Each of the specified classes are derived for each of the specified subjects.

```
command ::= ...
    | deriving instance derivingClass,* for term,*
```

Deriving Multiple Classes
After specifying multiple classes to derive for multiple types, as in this code:

##  10.4.1. Deriving Handlers[🔗](find/?domain=Verso.Genre.Manual.section&name=deriving-handlers "Permalink")
Instance deriving uses a table of _deriving handlers_ that maps type class names to metaprograms that derive instances for them. Deriving handlers may be added to the table using `[registerDerivingHandler](Type-Classes/Deriving-Instances/#Lean___Elab___registerDerivingHandler "Documentation for Lean.Elab.registerDerivingHandler")`, which should be called in an ``Lean.Parser.Command.initialize : command```initialize` block. Each deriving handler should have the type `[Array](Basic-Types/Arrays/#Array___mk "Documentation for Array") Name → CommandElabM [Bool](Basic-Types/Booleans/#Bool___false "Documentation for Bool")`. When a user requests that an instance of a class be derived, its registered handlers are called one at a time. They are provided with all of the names in the mutual block for which the instance is to be derived, and should either correctly derive an instance and return `[true](Basic-Types/Booleans/#Bool___false "Documentation for Bool.true")` or have no effect and return `[false](Basic-Types/Booleans/#Bool___false "Documentation for Bool.false")`. When a handler returns `[true](Basic-Types/Booleans/#Bool___false "Documentation for Bool.true")`, no further handlers are called.
Lean includes deriving handlers for the following classes:
  * `[BEq](Type-Classes/Basic-Classes/#BEq___mk "Documentation for BEq")`
  * `[DecidableEq](Type-Classes/Basic-Classes/#DecidableEq "Documentation for DecidableEq")`
  * `[Hashable](Type-Classes/Basic-Classes/#Hashable___mk "Documentation for Hashable")`
  * `[Inhabited](Type-Classes/Basic-Classes/#Inhabited___mk "Documentation for Inhabited")`
  * `[LawfulBEq](Type-Classes/Basic-Classes/#LawfulBEq___mk "Documentation for LawfulBEq")`
  * `[Nonempty](Type-Classes/Basic-Classes/#Nonempty___intro "Documentation for Nonempty")`
  * `[Ord](Type-Classes/Basic-Classes/#Ord___mk "Documentation for Ord")`
  * `[ReflBEq](Type-Classes/Basic-Classes/#ReflBEq___mk "Documentation for ReflBEq")`
  * `[Repr](Interacting-with-Lean/#Repr___mk "Documentation for Repr")`
  * `[SizeOf](The-Type-System/Inductive-Types/#SizeOf___mk "Documentation for SizeOf")`
  * `TypeName`

Registers a deriving handler for a class. This function should be called in an `initialize` block.
A `DerivingHandler` is called on the fully qualified names of all types it is running for. For example, `deriving instance Foo for Bar, Baz` invokes `fooHandler #[`Bar, `Baz]`.
Deriving Handlers
