#  12.1. Boxing[🔗](find/?domain=Verso.Genre.Manual.section&name=boxing "Permalink")
Lean values may be represented at runtime in two ways:
  * _Boxed_ values may be pointers to heap values or require shifting and masking.
  * _Unboxed_ values are immediately available.


Boxed values are either a pointer to an object, in which case the lowest-order bit is 0, or an immediate value, in which case the lowest-order bit is 1 and the value is found by shifting the representation to the right by one bit.
Types with an unboxed representation, such as `[UInt8](Basic-Types/Fixed-Precision-Integers/#UInt8___ofBitVec "Documentation for UInt8")` and [enum inductive](The-Type-System/Inductive-Types/#--tech-term-enum-inductive) types, are represented as the corresponding C types in contexts where the compiler can be sure that the value has said type. In some contexts, such as generic container types like `[Array](Basic-Types/Arrays/#Array___mk "Documentation for Array")`, otherwise-unboxed values must be boxed prior to storage. In other words, `[Bool.not](Basic-Types/Booleans/#Bool___not "Documentation for Bool.not")` is called with and returns unboxed `uint8_t` values because the [enum inductive](The-Type-System/Inductive-Types/#--tech-term-enum-inductive) type `[Bool](Basic-Types/Booleans/#Bool___false "Documentation for Bool")` has an unboxed representation, but the individual `[Bool](Basic-Types/Booleans/#Bool___false "Documentation for Bool")` values in an `[Array](Basic-Types/Arrays/#Array___mk "Documentation for Array") [Bool](Basic-Types/Booleans/#Bool___false "Documentation for Bool")` are boxed. A field of type `[Bool](Basic-Types/Booleans/#Bool___false "Documentation for Bool")` in an inductive type's constructor is represented unboxed, while `[Bool](Basic-Types/Booleans/#Bool___false "Documentation for Bool")`s stored in polymorphic fields that are instantiated as `[Bool](Basic-Types/Booleans/#Bool___false "Documentation for Bool")` are boxed.
