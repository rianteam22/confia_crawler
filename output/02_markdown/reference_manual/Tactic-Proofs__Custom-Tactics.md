#  14.8. Custom Tactics[🔗](find/?domain=Verso.Genre.Manual.section&name=custom-tactics "Permalink")
Tactics are productions in the syntax category `tactic`. Given the syntax of a tactic, the tactic interpreter is responsible for carrying out actions in the tactic monad `TacticM`, which is a wrapper around Lean's term elaborator that keeps track of the additional state needed to execute tactics. A custom tactic consists of an extension to the `tactic` category along with either:
  * a [macro](Notations-and-Macros/Macros/#--tech-term-Macros) that translates the new syntax into existing syntax, or
  * an elaborator that carries out `TacticM` actions to implement the tactic.


##  14.8.1. Tactic Macros[🔗](find/?domain=Verso.Genre.Manual.section&name=tactic-macros "Permalink")
The easiest way to define a new tactic is as a [macro](Notations-and-Macros/Macros/#--tech-term-Macros) that expands into already-existing tactics. Macro expansion is interleaved with tactic execution. The tactic interpreter first expands tactic macros just before they are to be interpreted. Because tactic macros are not fully expanded prior to running a tactic script, they can use recursion; as long as the recursive occurrence of the macro syntax is beneath a tactic that can be executed, there will not be an infinite chain of expansion.
Recursive tactic macro
This recursive implementation of a tactic akin to `[repeat](Tactic-Proofs/The-Tactic-Language/#repeat "Documentation for tactic")` is defined via macro expansion. When the argument `$t` fails, the recursive occurrence of `rep` is never invoked, and is thus never macro expanded.

Like other Lean macros, tactic macros are [hygienic](Notations-and-Macros/Macros/#--tech-term-hygienic). References to global names are resolved when the macro is defined, and names introduced by the tactic macro cannot capture names from its invocation site.
When defining a tactic macro, it's important to specify that the syntax being matched or constructed is for the syntax category `tactic`. Otherwise, the syntax will be interpreted as that of a term, which will match against or construct an incorrect AST for tactics.
###  14.8.1.1. Extensible Tactic Macros[🔗](find/?domain=Verso.Genre.Manual.section&name=tactic-macro-extension "Permalink")
Because macro expansion can fail, multiple macros can match the same syntax, allowing backtracking. Tactic macros take this further: even if a tactic macro expands successfully, if the expansion fails when interpreted, the tactic interpreter will attempt the next expansion. This is used to make a number of Lean's built-in tactics extensible—new behavior can be added to a tactic by adding a ``Lean.Parser.Command.macro_rules : command``[`macro_rules`](Notations-and-Macros/Macros/#Lean___Parser___Command___macro_rules) declaration.
Extending `[trivial](Tactic-Proofs/Tactic-Reference/#trivial "Documentation for tactic")`
The `[trivial](Tactic-Proofs/Tactic-Reference/#trivial "Documentation for tactic")`, which is used by many other tactics to quickly dispatch subgoals that are not worth bothering the user with, is designed to be extended through new macro expansions. Lean's default `trivial` can't solve `[IsEmpty](Tactic-Proofs/Custom-Tactics/#IsEmpty-_LPAR_in-Extending--trivial_RPAR_ "Definition of example") []` goals:

Expansion Backtracking
Macro expansion can induce backtracking when the failure arises from any part of the expanded syntax. An infix version of `[first](Tactic-Proofs/The-Tactic-Language/#first "Documentation for tactic")` can be defined by providing multiple expansions in separate ``Lean.Parser.Command.macro_rules : command``[`macro_rules`](Notations-and-Macros/Macros/#Lean___Parser___Command___macro_rules) declarations:

Multiple ``Lean.Parser.Command.macro_rules : command``[`macro_rules`](Notations-and-Macros/Macros/#Lean___Parser___Command___macro_rules) declarations are needed because each defines a pattern-matching function that will always take the first matching alternative. Backtracking is at the granularity of ``Lean.Parser.Command.macro_rules : command``[`macro_rules`](Notations-and-Macros/Macros/#Lean___Parser___Command___macro_rules) declarations, not their individual cases.
