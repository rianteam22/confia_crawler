#  3.1. Positive Numbers[🔗](find/?domain=Verso.Genre.Manual.section&name=positive-numbers "Permalink")
In some applications, only positive numbers make sense. For example, compilers and interpreters typically use one-indexed line and column numbers for source positions, and a datatype that represents only non-empty lists will never report a length of zero. Rather than relying on natural numbers, and littering the code with assertions that the number is not zero, it can be useful to design a datatype that represents only positive numbers.
One way to represent positive numbers is very similar to `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")`, except with `one` as the base case instead of `[zero](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat.zero in Lean Language Reference")`:
`inductive Pos : Type where   | one : Pos   | succ : Pos → Pos`

##  3.1.1. Classes and Instances[🔗](find/?domain=Verso.Genre.Manual.section&name=classes-and-instances "Permalink")
A type class consists of a name, some parameters, and a collection of _methods_. The parameters describe the types for which overloadable operations are being defined, and the methods are the names and type signatures of the overloadable operations. Once again, there is a terminology clash with object-oriented languages. In object-oriented programming, a method is essentially a function that is connected to a particular object in memory, with special access to the object's private state. Objects are interacted with via their methods. In Lean, the term “method” refers to an operation that has been declared to be overloadable, with no special connection to objects or values or private fields.
One way to overload addition is to define a type class named `Plus`, with an addition method named `plus`. Once an instance of `Plus` for `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` has been defined, it becomes possible to add two `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")`s using `Plus.plus`:
``8`[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") Plus.plus 5 3`
```
8
```

Adding more instances allows `Plus.plus` to take more types of arguments.
In the following type class declaration, `Plus` is the name of the class, `α : Type` is the only argument, and `plus : α → α → α` is the only method:
`class Plus (α : Type) where   plus : α → α → α`
This declaration says that there is a type class `Plus` that overloads operations with respect to a type `α`. In particular, there is one overloaded operation called `plus` that takes two `α`s and returns an `α`.
Type classes are first class, just as types are first class. In particular, a type class is another kind of type. The type of `Plus` is `Type → Type`, because it takes a type as an argument (`α`) and results in a new type that describes the overloading of `Plus`'s operation for `α`.
To overload `plus` for a particular type, write an instance:
`instance : Plus [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference") where   plus := [Nat.add](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___add "Documentation for Nat.add in Lean Language Reference")`
The colon after `instance` indicates that `Plus [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` is indeed a type. Each method of class `Plus` should be assigned a value using `:=`. In this case, there is only one method: `plus`.
By default, type class methods are defined in a namespace with the same name as the type class. It can be convenient to `[open](https://lean-lang.org/doc/reference/4.26.0/Namespaces-and-Sections/#Lean___Parser___Command___open "Documentation for syntax in Lean Language Reference")` the namespace so that users don't need to type the name of the class first. Parentheses in an `open` command indicate that only the indicated names from the namespace are to be made accessible:
`[open](https://lean-lang.org/doc/reference/4.26.0/Namespaces-and-Sections/#Lean___Parser___Command___open "Documentation for syntax in Lean Language Reference") Plus (plus)```8`[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") plus 5 3`
```
8
```

##  3.1.2. Overloaded Addition[🔗](find/?domain=Verso.Genre.Manual.section&name=overloaded-addition "Permalink")
Lean's built-in addition operator is syntactic sugar for a type class called `[HAdd](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd in Lean Language Reference")`, which flexibly allows the arguments to addition to have different types. `[HAdd](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd in Lean Language Reference")` is short for _heterogeneous addition_. For example, an `[HAdd](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd in Lean Language Reference")` instance can be written to allow a `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` to be added to a `[Float](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Floating-Point-Numbers/#Float-next "Documentation for Float in Lean Language Reference")`, resulting in a new `[Float](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Floating-Point-Numbers/#Float-next "Documentation for Float in Lean Language Reference")`. When a programmer writes `x + y`, it is interpreted as meaning `[HAdd.hAdd](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd.hAdd in Lean Language Reference") x y`.
While an understanding of the full generality of `[HAdd](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd in Lean Language Reference")` relies on features that are discussed in [another section in this chapter](Overloading-and-Type-Classes/Controlling-Instance-Search/#out-params), there is a simpler type class called `[Add](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Add___mk "Documentation for Add in Lean Language Reference")` that does not allow the types of the arguments to be mixed. The Lean libraries are set up so that an instance of `[Add](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Add___mk "Documentation for Add in Lean Language Reference")` will be found when searching for an instance of `[HAdd](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd in Lean Language Reference")` in which both arguments have the same type.
Defining an instance of `[Add](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Add___mk "Documentation for Add in Lean Language Reference") Pos` allows `Pos` values to use ordinary addition syntax:
`instance : [Add](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Add___mk "Documentation for Add in Lean Language Reference") Pos where   [add](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Add___mk "Documentation for Add.add in Lean Language Reference") := [Pos.plus](Overloading-and-Type-Classes/Positive-Numbers/#Pos___plus "Definition of example")``def fourteen : Pos := [seven](Overloading-and-Type-Classes/Positive-Numbers/#seven "Definition of example") + [seven](Overloading-and-Type-Classes/Positive-Numbers/#seven "Definition of example")`
##  3.1.3. Conversion to Strings[🔗](find/?domain=Verso.Genre.Manual.section&name=conversion-to-strings "Permalink")
Another useful built-in class is called `ToString`. Instances of `ToString` provide a standard way of converting values from a given type into strings. For example, a `ToString` instance is used when a value occurs in an interpolated string, and it determines how the `[IO.println](https://lean-lang.org/doc/reference/4.26.0/IO/Console-Output/#IO___println "Documentation for IO.println in Lean Language Reference")` function used at the [beginning of the description of ``](Hello___-World___/Running-a-Program/#running-a-program)`[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` will display a value.
For example, one way to convert a `Pos` into a `[String](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Strings/#String___mk "Documentation for String in Lean Language Reference")` is to reveal its inner structure. The function `posToString` takes a `[Bool](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Booleans/#Bool___false "Documentation for Bool in Lean Language Reference")` that determines whether to parenthesize uses of `[Pos.succ](Overloading-and-Type-Classes/Positive-Numbers/#Pos___succ "Definition of example")`, which should be `[true](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Booleans/#Bool___false "Documentation for Bool.true in Lean Language Reference")` in the initial call to the function and `[false](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Booleans/#Bool___false "Documentation for Bool.false in Lean Language Reference")` in all recursive calls.

##  3.1.4. Overloaded Multiplication[🔗](find/?domain=Verso.Genre.Manual.section&name=overloaded-multiplication "Permalink")
For multiplication, there is a type class called `[HMul](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HMul___mk "Documentation for HMul in Lean Language Reference")` that allows mixed argument types, just like `[HAdd](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd in Lean Language Reference")`. Just as `x + y` is interpreted as `[HAdd.hAdd](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd.hAdd in Lean Language Reference") x y`, `x * y` is interpreted as `[HMul.hMul](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HMul___mk "Documentation for HMul.hMul in Lean Language Reference") x y`. For the common case of multiplication of two arguments with the same type, a `[Mul](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Mul___mk "Documentation for Mul in Lean Language Reference")` instance suffices.
An instance of `[Mul](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Mul___mk "Documentation for Mul in Lean Language Reference")` allows ordinary multiplication syntax to be used with `Pos`:
`def Pos.mul : Pos → Pos → Pos   | Pos.one, k => k   | [Pos.succ](Overloading-and-Type-Classes/Positive-Numbers/#Pos___succ "Definition of example") n, k => n.[mul](Overloading-and-Type-Classes/Positive-Numbers/#Pos___mul "Definition of example") k + k  instance : [Mul](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Mul___mk "Documentation for Mul in Lean Language Reference") Pos where   [mul](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Mul___mk "Documentation for Mul.mul in Lean Language Reference") := [Pos.mul](Overloading-and-Type-Classes/Positive-Numbers/#Pos___mul "Definition of example")`
With this instance, multiplication works as expected:

##  3.1.5. Literal Numbers[🔗](find/?domain=Verso.Genre.Manual.section&name=literal-numbers "Permalink")
It is quite inconvenient to write out a sequence of constructors for positive numbers. One way to work around the problem would be to provide a function to convert a `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` into a `Pos`. However, this approach has downsides. First off, because `Pos` cannot represent `0`, the resulting function would either convert a `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` to a bigger number, or it would return `[Option](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Optional-Values/#Option___none "Documentation for Option in Lean Language Reference") Pos`. Neither is particularly convenient for users. Secondly, the need to call the function explicitly would make programs that use positive numbers much less convenient to write than programs that use `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")`. Having a trade-off between precise types and convenient APIs means that the precise types become less useful.
There are three type classes that are used to overload numeric literals: `Zero`, `One`, and `OfNat`. Because many types have values that are naturally written with `0`, the `Zero` class allow these specific values to be overridden. It is defined as follows:
`class Zero (α : Type) where   zero : α`
Because `0` is not a positive number, there should be no instance of `[Zero](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Zero___mk "Documentation for Zero in Lean Language Reference") Pos`.
Similarly, many types have values that are naturally written with `1`. The `One` class allows these to be overridden:
`class One (α : Type) where   one : α`
An instance of `One Pos` makes perfect sense:
`instance : One Pos where   one := Pos.one`
With this instance, `1` can be used for `Pos.one`:
``1`[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") (1 : Pos)`
```
1
```

In Lean, natural number literals are interpreted using a type class called `OfNat`:
`class OfNat (α : Type) (_ : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")) where   ofNat : α`
This type class takes two arguments: `α` is the type for which a natural number is overloaded, and the unnamed `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` argument is the actual literal number that was encountered in the program. The method `ofNat` is then used as the value of the numeric literal. Because the class contains the `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` argument, it becomes possible to define only instances for those values where the number makes sense.
`OfNat` demonstrates that the arguments to type classes do not need to be types. Because types in Lean are first-class participants in the language that can be passed as arguments to functions and given definitions with `def` and `abbrev`, there is no barrier that prevents non-type arguments in positions where a less-flexible language could not permit them. This flexibility allows overloaded operations to be provided for particular values as well as particular types. Additionally, it allows the Lean standard library to arrange for there to be a `[Zero](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Zero___mk "Documentation for Zero in Lean Language Reference") α` instance whenever there's an `[OfNat](https://lean-lang.org/doc/reference/4.26.0/Terms/Literals/#OfNat___mk "Documentation for OfNat in Lean Language Reference") α 0` instance, and vice versa. Similarly, an instance of `One α` implies an instance of `[OfNat](https://lean-lang.org/doc/reference/4.26.0/Terms/Literals/#OfNat___mk "Documentation for OfNat in Lean Language Reference") α 1`, just as an instance of `[OfNat](https://lean-lang.org/doc/reference/4.26.0/Terms/Literals/#OfNat___mk "Documentation for OfNat in Lean Language Reference") α 1` implies an instance of `One α`.
A sum type that represents natural numbers less than four can be defined as follows:
`inductive LT4 where   | zero   | one   | two   | three`
While it would not make sense to allow _any_ literal number to be used for this type, numbers less than four clearly make sense:

##  3.1.6. Exercises[🔗](find/?domain=Verso.Genre.Manual.section&name=positive-numbers-exercises "Permalink")
###  3.1.6.1. Another Representation[🔗](find/?domain=Verso.Genre.Manual.section&name=positive-numbers-another-representation "Permalink")
An alternative way to represent a positive number is as the successor of some `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")`. Replace the definition of `Pos` with a structure whose constructor is named `succ` that contains a `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")`:
`structure Pos where   succ ::   pred : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")`
Define instances of `[Add](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Add___mk "Documentation for Add in Lean Language Reference")`, `[Mul](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Mul___mk "Documentation for Mul in Lean Language Reference")`, `ToString`, and `[OfNat](https://lean-lang.org/doc/reference/4.26.0/Terms/Literals/#OfNat___mk "Documentation for OfNat in Lean Language Reference")` that allow this version of `Pos` to be used conveniently.
###  3.1.6.2. Even Numbers[🔗](find/?domain=Verso.Genre.Manual.section&name=even-numbers-ex "Permalink")
Define a datatype that represents only even numbers. Define instances of `[Add](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Add___mk "Documentation for Add in Lean Language Reference")`, `[Mul](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#Mul___mk "Documentation for Mul in Lean Language Reference")`, and `ToString` that allow it to be used conveniently. `[OfNat](https://lean-lang.org/doc/reference/4.26.0/Terms/Literals/#OfNat___mk "Documentation for OfNat in Lean Language Reference")` requires a feature that is introduced in [the next section](Overloading-and-Type-Classes/Type-Classes-and-Polymorphism/#tc-polymorphism).
###  3.1.6.3. HTTP Requests[🔗](find/?domain=Verso.Genre.Manual.section&name=http-request-ex "Permalink")
An HTTP request begins with an identification of a HTTP method, such as `GET` or `POST`, along with a URI and an HTTP version. Define an inductive type that represents an interesting subset of the HTTP methods, and a structure that represents HTTP responses. Responses should have a `ToString` instance that makes it possible to debug them. Use a type class to associate different `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` actions with each HTTP method, and write a test harness as an `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action that calls each method and prints the result.
