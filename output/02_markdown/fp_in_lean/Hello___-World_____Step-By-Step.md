#  2.2. Step By Step[🔗](find/?domain=Verso.Genre.Manual.section&name=step-by-step "Permalink")
A `[do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")` block can be executed one line at a time. Start with the program from the prior section:

##  2.2.1. Standard IO[🔗](find/?domain=Verso.Genre.Manual.section&name=stdio "Permalink")
The first line is `let stdin ← [IO.getStdin](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___getStdin "Documentation for IO.getStdin in Lean Language Reference")`, while the remainder is:
`let stdout ← [IO.getStdout](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___getStdout "Documentation for IO.getStdout in Lean Language Reference") stdout.[putStrLn](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___putStrLn "Documentation for IO.FS.Stream.putStrLn in Lean Language Reference") "How would you like to be addressed?" let input ← stdin.[getLine](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream.getLine in Lean Language Reference") let name := input.[dropRightWhile](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Strings/#String___dropRightWhile "Documentation for String.dropRightWhile in Lean Language Reference") [Char.isWhitespace](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Characters/#Char___isWhitespace "Documentation for Char.isWhitespace in Lean Language Reference") stdout.[putStrLn](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___putStrLn "Documentation for IO.FS.Stream.putStrLn in Lean Language Reference") s!"Hello, {name}!"`
To execute a `let` statement that uses a `←`, start by evaluating the expression to the right of the arrow (in this case, `[IO.getStdin](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___getStdin "Documentation for IO.getStdin in Lean Language Reference")`). Because this expression is just a variable, its value is looked up. The resulting value is a built-in primitive `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action. The next step is to execute this `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action, resulting in a value that represents the standard input stream, which has type `[IO.FS.Stream](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream in Lean Language Reference")`. Standard input is then associated with the name to the left of the arrow (here `stdin`) for the remainder of the `[do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")` block.
Executing the second line, `let stdout ← [IO.getStdout](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___getStdout "Documentation for IO.getStdout in Lean Language Reference")`, proceeds similarly. First, the expression `[IO.getStdout](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___getStdout "Documentation for IO.getStdout in Lean Language Reference")` is evaluated, yielding an `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action that will return the standard output. Next, this action is executed, actually returning the standard output. Finally, this value is associated with the name `stdout` for the remainder of the `[do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")` block.
##  2.2.2. Asking a Question[🔗](find/?domain=Verso.Genre.Manual.section&name=asking-a-question "Permalink")
Now that `stdin` and `stdout` have been found, the remainder of the block consists of a question and an answer:
`stdout.[putStrLn](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___putStrLn "Documentation for IO.FS.Stream.putStrLn in Lean Language Reference") "How would you like to be addressed?" let input ← stdin.[getLine](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream.getLine in Lean Language Reference") let name := input.[dropRightWhile](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Strings/#String___dropRightWhile "Documentation for String.dropRightWhile in Lean Language Reference") [Char.isWhitespace](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Characters/#Char___isWhitespace "Documentation for Char.isWhitespace in Lean Language Reference") stdout.[putStrLn](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___putStrLn "Documentation for IO.FS.Stream.putStrLn in Lean Language Reference") s!"Hello, {name}!"`
The first statement in the block, `stdout.[putStrLn](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___putStrLn "Documentation for IO.FS.Stream.putStrLn in Lean Language Reference") "How would you like to be addressed?"`, consists of an expression. To execute an expression, it is first evaluated. In this case, `[IO.FS.Stream.putStrLn](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___putStrLn "Documentation for IO.FS.Stream.putStrLn in Lean Language Reference")` has type `[IO.FS.Stream](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream in Lean Language Reference") → [String](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Strings/#String___mk "Documentation for String in Lean Language Reference") → [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference")`. This means that it is a function that accepts a stream and a string, returning an `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action. The expression uses [accessor notation](Getting-to-Know-Lean/Structures/#behind-the-scenes) for a function call. This function is applied to two arguments: the standard output stream and a string. The value of the expression is an `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action that will write the string and a newline character to the output stream. Having found this value, the next step is to execute it, which causes the string and newline to actually be written to `stdout`. Statements that consist only of expressions do not introduce any new variables.
The next statement in the block is `let input ← stdin.[getLine](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream.getLine in Lean Language Reference")`. `IO.FS.Stream.getLine` has type `[IO.FS.Stream](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream in Lean Language Reference") → [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [String](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Strings/#String___mk "Documentation for String in Lean Language Reference")`, which means that it is a function from a stream to an `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action that will return a string. Once again, this is an example of accessor notation. This `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action is executed, and the program waits until the user has typed a complete line of input. Assume the user writes “`David`”. The resulting line (`"David\n"`) is associated with `input`, where the escape sequence `\n` denotes the newline character.
`let name := input.[dropRightWhile](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Strings/#String___dropRightWhile "Documentation for String.dropRightWhile in Lean Language Reference") [Char.isWhitespace](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Characters/#Char___isWhitespace "Documentation for Char.isWhitespace in Lean Language Reference") stdout.[putStrLn](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___putStrLn "Documentation for IO.FS.Stream.putStrLn in Lean Language Reference") s!"Hello, {name}!"`

##  2.2.3. Greeting the User[🔗](find/?domain=Verso.Genre.Manual.section&name=greeting "Permalink")
All that remains to be executed in the `[do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")` block is a single statement:
`stdout.[putStrLn](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___putStrLn "Documentation for IO.FS.Stream.putStrLn in Lean Language Reference") s!"Hello, {name}!"`
The string argument to `[putStrLn](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___putStrLn "Documentation for IO.FS.Stream.putStrLn in Lean Language Reference")` is constructed via string interpolation, yielding the string `"Hello, David!"`. Because this statement is an expression, it is evaluated to yield an `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action that will print this string with a newline to standard output. Once the expression has been evaluated, the resulting `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action is executed, resulting in the greeting.
##  2.2.4. `IO` Actions as Values[🔗](find/?domain=Verso.Genre.Manual.section&name=actions-as-values "Permalink")
In the above description, it can be difficult to see why the distinction between evaluating expressions and executing `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` actions is necessary. After all, each action is executed immediately after it is produced. Why not simply carry out the effects during evaluation, as is done in other languages?
The answer is twofold. First off, separating evaluation from execution means that programs must be explicit about which functions can have side effects. Because the parts of the program that do not have effects are much more amenable to mathematical reasoning, whether in the heads of programmers or using Lean's facilities for formal proof, this separation can make it easier to avoid bugs. Secondly, not all `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` actions need be executed at the time that they come into existence. The ability to mention an action without carrying it out allows ordinary functions to be used as control structures.
For example, the function `twice` takes an `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action as its argument, returning a new action that will execute the argument action twice.
`def twice (action : [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference")) : [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference") := [do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")   action   action`
Executing
`twice ([IO.println](https://lean-lang.org/doc/reference/4.26.0/IO/Console-Output/#IO___println "Documentation for IO.println in Lean Language Reference") "shy")`
results in

```
shy
shy

```

being printed. This can be generalized to a version that runs the underlying action any number of times:
`def nTimes (action : [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference")) : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference") → [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference")   | 0 => [pure](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/#Pure___mk "Documentation for Pure.pure in Lean Language Reference") ()   | n + 1 => [do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")     action     nTimes action n`
In the base case for `[Nat.zero](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat.zero in Lean Language Reference")`, the result is `[pure](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/#Pure___mk "Documentation for Pure.pure in Lean Language Reference") ()`. The function `[pure](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/#Pure___mk "Documentation for Pure.pure in Lean Language Reference")` creates an `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` action that has no side effects, but returns `[pure](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/#Pure___mk "Documentation for Pure.pure in Lean Language Reference")`'s argument, which in this case is the constructor for `[Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference")`. As an action that does nothing and returns nothing interesting, `[pure](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/#Pure___mk "Documentation for Pure.pure in Lean Language Reference") ()` is at the same time utterly boring and very useful. In the recursive step, a `[do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")` block is used to create an action that first executes `action` and then executes the result of the recursive call. Executing ``Hello Hello Hello `[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") nTimes ([IO.println](https://lean-lang.org/doc/reference/4.26.0/IO/Console-Output/#IO___println "Documentation for IO.println in Lean Language Reference") "Hello") 3` causes the following output:

```
Hello
Hello
Hello

```

In addition to using functions as control structures, the fact that `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` actions are first-class values means that they can be saved in data structures for later execution. For instance, the function `countdown` takes a `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` and returns a list of unexecuted `[IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference")` actions, one for each `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")`:
`def countdown : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference") → [List](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___nil "Documentation for List in Lean Language Reference") ([IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference"))   | 0 => [[IO.println](https://lean-lang.org/doc/reference/4.26.0/IO/Console-Output/#IO___println "Documentation for IO.println in Lean Language Reference") "Blast off!"]   | n + 1 => [IO.println](https://lean-lang.org/doc/reference/4.26.0/IO/Console-Output/#IO___println "Documentation for IO.println in Lean Language Reference") s!"{n + 1}" :: countdown n`
This function has no side effects, and does not print anything. For example, it can be applied to an argument, and the length of the resulting list of actions can be checked:
`def from5 : [List](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___nil "Documentation for List in Lean Language Reference") ([IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference")) := countdown 5`
This list contains six elements (one for each number, plus a `"Blast off!"` action for zero):
`[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") [from5](Hello___-World___/Step-By-Step/#from5 "Definition of example").[length](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___length "Documentation for List.length in Lean Language Reference")`
```

```

The function `runActions` takes a list of actions and constructs a single action that runs them all in order:
`def runActions : [List](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___nil "Documentation for List in Lean Language Reference") ([IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference")) → [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference")   | [] => [pure](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/#Pure___mk "Documentation for Pure.pure in Lean Language Reference") ()   | act :: actions => [do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")     act     runActions actions`
Its structure is essentially the same as that of `nTimes`, except instead of having one action that is executed for each `[Nat.succ](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat.succ in Lean Language Reference")`, the action under each `[List.cons](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___nil "Documentation for List.cons in Lean Language Reference")` is to be executed. Similarly, `runActions` does not itself run the actions. It creates a new action that will run them, and that action must be placed in a position where it will be executed as a part of `main`:
`def main : [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference") := runActions [from5](Hello___-World___/Step-By-Step/#from5 "Definition of example")`
Running this program results in the following output:
`countdown``5 4 3 2 1 Blast off! `
