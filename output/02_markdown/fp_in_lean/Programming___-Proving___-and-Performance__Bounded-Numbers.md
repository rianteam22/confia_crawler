#  8.5. Bounded Numbers[🔗](find/?domain=Verso.Genre.Manual.section&name=Fin "Permalink")
The `[GetElem](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#GetElem___mk "Documentation for GetElem in Lean Language Reference")` instance for `[Array](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Arrays/#Array___mk "Documentation for Array in Lean Language Reference")` and `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` requires a proof that the provided `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` is smaller than the array. In practice, these proofs often end up being passed to functions along with the indices. Rather than passing an index and a proof separately, a type called `Fin` can be used to bundle up the index and the proof into a single value. This can make code easier to read.
The type `[Fin](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Finite-Natural-Numbers/#Fin___mk "Documentation for Fin in Lean Language Reference") n` represents numbers that are strictly less than `n`. In other words, `[Fin](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Finite-Natural-Numbers/#Fin___mk "Documentation for Fin in Lean Language Reference") 3` describes `0`, `1`, and `2`, while `[Fin](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Finite-Natural-Numbers/#Fin___mk "Documentation for Fin in Lean Language Reference") 0` has no values at all. The definition of `Fin` resembles `[Subtype](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Subtypes/#Subtype___mk "Documentation for Subtype in Lean Language Reference")`, as a `[Fin](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Finite-Natural-Numbers/#Fin___mk "Documentation for Fin in Lean Language Reference") n` is a structure that contains a `[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")` and a proof that it is less than `n`:
`structure Fin (n : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")) where   val  : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")   isLt : [LT.lt](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#LT___mk "Documentation for LT.lt in Lean Language Reference") val n`
Lean includes instances of `ToString` and `[OfNat](https://lean-lang.org/doc/reference/4.26.0/Terms/Literals/#OfNat___mk "Documentation for OfNat in Lean Language Reference")` that allow `Fin` values to be conveniently used as numbers. In other words, the output of `[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") (5 : [Fin](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Finite-Natural-Numbers/#Fin___mk "Documentation for Fin in Lean Language Reference") 8)` is `5`, rather than something like `{val := 5, isLt := _}`.
Instead of failing when the provided number is larger than the bound, the `[OfNat](https://lean-lang.org/doc/reference/4.26.0/Terms/Literals/#OfNat___mk "Documentation for OfNat in Lean Language Reference")` instance for `Fin` returns a value modulo the bound. This means that `[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") (45 : [Fin](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Finite-Natural-Numbers/#Fin___mk "Documentation for Fin in Lean Language Reference") 10)` results in `5` rather than a compile-time error.
In a return type, a `Fin` returned as a found index makes its connection to the data structure in which it was found more clear. The `Array.find` in the [previous section](Programming___-Proving___-and-Performance/Arrays-and-Termination/#proving-termination) returns an index that the caller cannot immediately use to perform lookups into the array, because the information about its validity has been lost. ##  8.5.1. Exercise[🔗](find/?domain=Verso.Genre.Manual.section&name=Fin-exercises "Permalink")
Write a function `Fin.next? : [Fin](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Finite-Natural-Numbers/#Fin___mk "Documentation for Fin in Lean Language Reference") n → [Option](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Optional-Values/#Option___none "Documentation for Option in Lean Language Reference") ([Fin](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Finite-Natural-Numbers/#Fin___mk "Documentation for Fin in Lean Language Reference") n)` that returns the next largest `[Fin](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Finite-Natural-Numbers/#Fin___mk "Documentation for Fin in Lean Language Reference")` when it would be in bounds, or `[none](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Optional-Values/#Option___none "Documentation for Option.none in Lean Language Reference")` if not. Check that
``[some](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Optional-Values/#Option___none "Documentation for Option.some in Lean Language Reference") 4`[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") (3 : [Fin](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Finite-Natural-Numbers/#Fin___mk "Documentation for Fin in Lean Language Reference") 8).next?`
outputs

```
[some](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Optional-Values/#Option___none "Documentation for Option.some in Lean Language Reference") 4
```

