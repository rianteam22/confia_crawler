#  6.5. Additional Conveniences[🔗](find/?domain=Verso.Genre.Manual.section&name=monad-transformer-conveniences "Permalink")
##  6.5.1. Pipe Operators[🔗](find/?domain=Verso.Genre.Manual.section&name=pipe-operators "Permalink")
Functions are normally written before their arguments. When reading a program from left to right, this promotes a view in which the function's _output_ is paramount—the function has a goal to achieve (that is, a value to compute), and it receives arguments to support it in this process. But some programs are easier to understand in terms of an input that is successively refined to produce the output. For these situations, Lean provides a _pipeline_ operator which is similar to the that provided by F#. Pipeline operators are useful in the same situations as Clojure's threading macros.
The pipeline `E₁ |> E₂` is short for `E₂ E₁`. For example, evaluating:
``"(some 5)"`[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") [some](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Optional-Values/#Option___none "Documentation for Option.some in Lean Language Reference") 5 |> toString`
results in:

```
"(some 5)"
```

While this change of emphasis can make some programs more convenient to read, pipelines really come into their own when they contain many components.
With the definition:
`def times3 (n : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")) : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference") := n * 3`
the following pipeline:
``"It is 15"`[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") 5 |> [times3](Monad-Transformers/Additional-Conveniences/#PipelineEx___times3 "Definition of example") |> toString |> ("It is " ++ ·)`
yields:

```
"It is 15"
```

More generally, a series of pipelines `E₁ |> E₂ |> E₃ |> E₄` is short for nested function applications `E₄ (E₃ (E₂ E₁))`.
Pipelines may also be written in reverse. In this case, they do not place the subject of data transformation first; however, in cases where many nested parentheses pose a challenge for readers, they can clarify the steps of application. The prior example could be equivalently written as:
``"It is 15"`[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") ("It is " ++ ·) <| toString <| [times3](Monad-Transformers/Additional-Conveniences/#PipelineEx___times3 "Definition of example") <| 5`
which is short for:
``"It is 15"`[#eval](https://lean-lang.org/doc/reference/4.26.0/Interacting-with-Lean/#Lean___Parser___Command___eval "Documentation for syntax in Lean Language Reference") ("It is " ++ ·) (toString ([times3](Monad-Transformers/Additional-Conveniences/#PipelineEx___times3 "Definition of example") 5))`
Lean's method dot notation that uses the name of the type before the dot to resolve the namespace of the operator after the dot serves a similar purpose to pipelines. Even without the pipeline operator, it is possible to write `[1, 2, 3].[reverse](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___reverse "Documentation for List.reverse in Lean Language Reference")` instead of `[List.reverse](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___reverse "Documentation for List.reverse in Lean Language Reference") [1, 2, 3]`. However, the pipeline operator is also useful for dotted functions when using many of them. `([1, 2, 3].[reverse](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___reverse "Documentation for List.reverse in Lean Language Reference").[drop](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___drop "Documentation for List.drop in Lean Language Reference") 1).[reverse](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___reverse "Documentation for List.reverse in Lean Language Reference")` can also be written as `[1, 2, 3] |> [List.reverse](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___reverse "Documentation for List.reverse in Lean Language Reference") |> [List.drop](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___drop "Documentation for List.drop in Lean Language Reference") 1 |> [List.reverse](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___reverse "Documentation for List.reverse in Lean Language Reference")`. This version avoids having to parenthesize expressions simply because they accept arguments, and it recovers the convenience of a chain of method calls in languages like Kotlin or C#. However, it still requires the namespace to be provided by hand. As a final convenience, Lean provides the “pipeline dot” operator, which groups functions like the pipeline but uses the name of the type to resolve namespaces. With “pipeline dot”, the example can be rewritten to `[1, 2, 3] |>.[reverse](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___reverse "Documentation for List.reverse in Lean Language Reference") |>.[drop](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___drop "Documentation for List.drop in Lean Language Reference") 1 |>.[reverse](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Linked-Lists/#List___reverse "Documentation for List.reverse in Lean Language Reference")`.
##  6.5.2. Infinite Loops[🔗](find/?domain=Verso.Genre.Manual.section&name=infinite-loops "Permalink")
Within a `do`-block, the `repeat` keyword introduces an infinite loop. For example, a program that spams the string `"Spam!"` can use it:
`def spam : [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference") := [do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")   repeat [IO.println](https://lean-lang.org/doc/reference/4.26.0/IO/Console-Output/#IO___println "Documentation for IO.println in Lean Language Reference") "Spam!"`
A `repeat` loop supports `break` and `continue`, just like `for` loops.
The `dump` function from the [implementation of `feline`](Hello___-World___/Worked-Example___--cat/#streams) uses a recursive function to run forever:
`partial def dump (stream : [IO.FS.Stream](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream in Lean Language Reference")) : [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference") := [do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")   let buf ← stream.[read](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream.read in Lean Language Reference") [bufsize](Hello___-World___/Worked-Example___--cat/#bufsize "Definition of example")   if buf.isEmpty then     [pure](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/#Pure___mk "Documentation for Pure.pure in Lean Language Reference") ()   else     let stdout ← [IO.getStdout](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___getStdout "Documentation for IO.getStdout in Lean Language Reference")     stdout.[write](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream.write in Lean Language Reference") buf     dump stream`
This function can be greatly shortened using `repeat`:
`def dump (stream : [IO.FS.Stream](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream in Lean Language Reference")) : [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference") := [do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")   let stdout ← [IO.getStdout](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___getStdout "Documentation for IO.getStdout in Lean Language Reference")   repeat do     let buf ← stream.[read](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream.read in Lean Language Reference") [bufsize](Hello___-World___/Worked-Example___--cat/#bufsize "Definition of example")     if buf.isEmpty then break     stdout.[write](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream.write in Lean Language Reference") buf`
Neither `spam` nor `dump` need to be declared as `partial` because they are not themselves infinitely recursive. Instead, `repeat` makes use of a type whose `ForM` instance is `partial`. Partiality does not “infect” calling functions.
##  6.5.3. While Loops[🔗](find/?domain=Verso.Genre.Manual.section&name=while-loops "Permalink")
When programming with local mutability, `while` loops can be a convenient alternative to `repeat` with an `if`-guarded `break`:
`def dump (stream : [IO.FS.Stream](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream in Lean Language Reference")) : [IO](https://lean-lang.org/doc/reference/4.26.0/IO/Logical-Model/#IO "Documentation for IO in Lean Language Reference") [Unit](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/The-Unit-Type/#Unit "Documentation for Unit in Lean Language Reference") := [do](https://lean-lang.org/doc/reference/4.26.0/Functors___-Monads-and--do--Notation/Syntax/#Lean___Parser___Term___do "Documentation for syntax in Lean Language Reference")   let stdout ← [IO.getStdout](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___getStdout "Documentation for IO.getStdout in Lean Language Reference")   let mut buf ← stream.[read](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream.read in Lean Language Reference") [bufsize](Hello___-World___/Worked-Example___--cat/#bufsize "Definition of example")   while [not](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Booleans/#Bool___not "Documentation for Bool.not in Lean Language Reference") buf.isEmpty do     stdout.[write](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream.write in Lean Language Reference") buf     buf ← stream.[read](https://lean-lang.org/doc/reference/4.26.0/IO/Files___-File-Handles___-and-Streams/#IO___FS___Stream___mk "Documentation for IO.FS.Stream.read in Lean Language Reference") [bufsize](Hello___-World___/Worked-Example___--cat/#bufsize "Definition of example")`
Behind the scenes, `while` is just a simpler notation for `repeat`.
