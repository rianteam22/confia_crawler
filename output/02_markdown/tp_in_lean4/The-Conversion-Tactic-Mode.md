#  11. The Conversion Tactic Mode[🔗](find/?domain=Verso.Genre.Manual.section&name=conv "Permalink")
Inside a tactic block, one can use the keyword `conv` to enter _conversion mode_. This mode allows to travel inside assumptions and goals, even inside function abstractions and dependent arrows, to apply rewriting or simplifying steps.
##  11.1. Basic navigation and rewriting[🔗](find/?domain=Verso.Genre.Manual.section&name=basic-navigation-and-rewriting "Permalink")
As a first example, let us prove example `(a b c : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")) : a * (b * c) = a * (c * b)` (examples in this file are somewhat artificial since other tactics could finish them immediately). The naive first attempt is to enter tactic mode and try `[rw](https://lean-lang.org/doc/reference/4.26.0/Tactic-Proofs/Tactic-Reference/#rw "Documentation for tactic in Lean Language Reference") [Nat.mul_comm]`. But this transforms the goal into `b [*](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HMul___mk "Documentation for HMul.hMul in Lean Language Reference") c [*](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HMul___mk "Documentation for HMul.hMul in Lean Language Reference") a [=](https://lean-lang.org/doc/reference/4.26.0/Basic-Propositions/Propositional-Equality/#Eq___refl "Documentation for Eq in Lean Language Reference") a [*](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HMul___mk "Documentation for HMul.hMul in Lean Language Reference") [(](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HMul___mk "Documentation for HMul.hMul in Lean Language Reference")c [*](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HMul___mk "Documentation for HMul.hMul in Lean Language Reference") b[)](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HMul___mk "Documentation for HMul.hMul in Lean Language Reference")`, after commuting the very first multiplication appearing in the term. The above snippet shows three navigation commands:
  * `lhs` navigates to the left-hand side of a relation (equality, in this case). There is also a `rhs` to navigate to the right-hand side.
  * `congr` creates as many targets as there are (nondependent and explicit) arguments to the current head function (here the head function is multiplication).
  * `rfl` closes target using reflexivity.


Once arrived at the relevant target, we can use `[rw](https://lean-lang.org/doc/reference/4.26.0/Tactic-Proofs/Tactic-Reference/#rw "Documentation for tactic in Lean Language Reference")` as in normal tactic mode.
The second main reason to use conversion mode is to rewrite under binders. Suppose we want to prove example `(fun x : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference") => 0 + x) = (fun x => x)`. The naive first attempt is to enter tactic mode and try `rw [Nat.zero_add]`. But this fails with a frustrating

```
error: tactic 'rewrite' failed, did not find instance of the pattern
       in the target expression
  0 + ?n
⊢ (fun x => 0 + x) = fun x => x

```

The solution is:
`example : (fun x : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference") => 0 + x) = (fun x => x) :=  by⊢ (fun x => 0 [+](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd.hAdd in Lean Language Reference") x) [=](https://lean-lang.org/doc/reference/4.26.0/Basic-Propositions/Propositional-Equality/#Eq___refl "Documentation for Eq in Lean Language Reference") fun x => x   conv =>| (fun x => 0 [+](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd.hAdd in Lean Language Reference") x) [=](https://lean-lang.org/doc/reference/4.26.0/Basic-Propositions/Propositional-Equality/#Eq___refl "Documentation for Eq in Lean Language Reference") fun x => x     lhs| fun x => 0 [+](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd.hAdd in Lean Language Reference") x     intro xhx:[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")| 0 [+](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd.hAdd in Lean Language Reference") x     rw [Nat.zero_add]hx:[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")| x ```
where `intro x` is the navigation command entering inside the `fun` binder. Note that this example is somewhat artificial, one could also do:
`example : (fun x : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference") => 0 + x) = (fun x => x) := by⊢ (fun x => 0 [+](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd.hAdd in Lean Language Reference") x) [=](https://lean-lang.org/doc/reference/4.26.0/Basic-Propositions/Propositional-Equality/#Eq___refl "Documentation for Eq in Lean Language Reference") fun x => x   [funext](https://lean-lang.org/doc/reference/4.26.0/Tactic-Proofs/Tactic-Reference/#funext-next "Documentation for tactic in Lean Language Reference") xhx:[Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference")⊢ 0 [+](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd.hAdd in Lean Language Reference") x [=](https://lean-lang.org/doc/reference/4.26.0/Basic-Propositions/Propositional-Equality/#Eq___refl "Documentation for Eq in Lean Language Reference") x; [rw](https://lean-lang.org/doc/reference/4.26.0/Tactic-Proofs/Tactic-Reference/#rw "Documentation for tactic in Lean Language Reference") [Nat.zero_add]All goals completed! 🐙 ```
or just
`example : (fun x : [Nat](https://lean-lang.org/doc/reference/4.26.0/Basic-Types/Natural-Numbers/#Nat___zero "Documentation for Nat in Lean Language Reference") => 0 + x) = (fun x => x) := by⊢ (fun x => 0 [+](https://lean-lang.org/doc/reference/4.26.0/Type-Classes/Basic-Classes/#HAdd___mk "Documentation for HAdd.hAdd in Lean Language Reference") x) [=](https://lean-lang.org/doc/reference/4.26.0/Basic-Propositions/Propositional-Equality/#Eq___refl "Documentation for Eq in Lean Language Reference") fun x => x   [simp](https://lean-lang.org/doc/reference/4.26.0/Tactic-Proofs/Tactic-Reference/#simp "Documentation for tactic in Lean Language Reference")All goals completed! 🐙 ```
`conv` can also rewrite a hypothesis `h` from the local context, using `conv at`` h`.
